# Documentation

This folder is the index for the working documentation that sits next to the code. It is meant to stay short and readable. The goal is not to duplicate every GitHub issue in prose, but to keep the core architectural decisions and repository conventions in one place so the codebase can be understood without reconstructing context from project boards.

Start with [architecture.md](/U:/Personal/ParetoProof/docs/architecture.md) if you want the product and service model, then [repository-layout.md](/U:/Personal/ParetoProof/docs/repository-layout.md) if you want the codebase shape. [deployment-baseline.md](/U:/Personal/ParetoProof/docs/deployment-baseline.md) describes how the repo is prepared for Pages, Railway, and worker-image builds without overcommitting early implementation details. [operations-baseline.md](/U:/Personal/ParetoProof/docs/operations-baseline.md) records the service ownership, hostname, and secret model that the current roadmap relies on. [data-baseline.md](/U:/Personal/ParetoProof/docs/data-baseline.md) captures the first database entity and branch strategy decisions. [project-management.md](/U:/Personal/ParetoProof/docs/project-management.md) explains how scoping issues and execution issues are meant to move through GitHub. The accepted implementation stack decision is recorded in [adrs/0001-mvp-implementation-stack.md](/U:/Personal/ParetoProof/docs/adrs/0001-mvp-implementation-stack.md) and the hosting choice in [adrs/0002-mvp-hosting-decision.md](/U:/Personal/ParetoProof/docs/adrs/0002-mvp-hosting-decision.md).
