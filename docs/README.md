# Documentation

This folder is the index for the working documentation that sits next to the code. It is meant to stay short and readable. The goal is not to duplicate every GitHub issue in prose, but to keep the core architectural decisions and repository conventions in one place so the codebase can be understood without reconstructing context from project boards.

Start with [architecture.md](architecture.md) if you want the product and service model, then [repository-layout.md](repository-layout.md) if you want the codebase shape. [deployment-baseline.md](deployment-baseline.md) describes how the repo is prepared for Pages, Railway, and worker-image builds, and now also records the baseline rollback path for each deployable surface without overcommitting early implementation details. [staging-promotion-baseline.md](staging-promotion-baseline.md) defines how one merged SHA enters staging, what gets validated there, and how a human promotes the same anchored revision toward production. [production-release-checklist-baseline.md](production-release-checklist-baseline.md) defines the final production checklist, approval gates, and rollback handoff required before a staged candidate may become a live release. [tls-hostname-baseline.md](tls-hostname-baseline.md) defines which platform terminates public TLS for each ParetoProof hostname and whether Cloudflare should proxy or pass records through. [math-surface-mvp-baseline.md](math-surface-mvp-baseline.md) decides whether `math.paretoproof.com` is part of the MVP product map and what math-specific work stays on the apex or portal surfaces instead. [product-surface-boundary-baseline.md](product-surface-boundary-baseline.md) defines which workflows, route families, and user roles belong on the public apex, auth-entry, and portal surfaces during MVP. [runtime-env-contract-baseline.md](runtime-env-contract-baseline.md) defines required versus optional environment variables by app and runtime mode, plus where startup validation should happen. [math-artifact-types-baseline.md](math-artifact-types-baseline.md) defines the in-scope mathematical artifact taxonomy and how artifact types map onto benchmark tasks. [source-problem-selection-baseline.md](source-problem-selection-baseline.md) defines which source families ParetoProof may curate, how contamination-sensitive source exposure should be labeled, and what provenance record each selected item must preserve. [benchmark-curation-criteria-baseline.md](benchmark-curation-criteria-baseline.md) defines the minimum difficulty, novelty, contamination, theorem-quality, and Lean-suitability bar for keeping a selected item as an active benchmark candidate. [public-disclosure-holdout-policy-baseline.md](public-disclosure-holdout-policy-baseline.md) defines the separation between public benchmark material and private hold-out material, the allowed public claims for each, and the curation-discussion constraints that follow from that split. [problem-statement-metadata-baseline.md](problem-statement-metadata-baseline.md) defines the canonical benchmark item statement pair, required provenance and classification metadata, and the expected-output-type field contract. [gold-reference-solution-policy-baseline.md](gold-reference-solution-policy-baseline.md) defines when benchmark items require internal or public gold/reference artifacts, how official upstream formalizations may be adopted, and how withheld references behave in MVP. [lean-formalization-standards-baseline.md](lean-formalization-standards-baseline.md) defines the repository-owned Lean naming, import, theorem-shape, and proof-completion rules for benchmark-owned formalizations. [docker-image-baseline.md](docker-image-baseline.md) defines the offline `firstproof/Problem9` devbox and execution-image policy, runtime-tool boundary, and never-in-image secret rules. [lean-mathlib-version-baseline.md](lean-mathlib-version-baseline.md) defines the canonical Lean lanes, Mathlib pinning rules, mixed-version handling, and version-change policy for the offline `firstproof/Problem9` slice. [problem9-run-bundle-baseline.md](problem9-run-bundle-baseline.md) defines the canonical offline run bundle, verifier verdict subset, ingest-critical files, and digest boundary for the Problem 9 slice.
