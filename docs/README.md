# Documentation

This folder is the index for the working documentation that sits next to the code. It is meant to stay short and readable. The goal is not to duplicate every GitHub issue in prose, but to keep the core architectural decisions and repository conventions in one place so the codebase can be understood without reconstructing context from project boards.

Start with [architecture.md](architecture.md) if you want the product and service model, then [repository-layout.md](repository-layout.md) if you want the codebase shape. [deployment-baseline.md](deployment-baseline.md) describes how the repo is prepared for Pages, Railway, and worker-image builds, and now also records the baseline rollback path for each deployable surface without overcommitting early implementation details. [docker-image-baseline.md](docker-image-baseline.md) defines the pinned MVP API/worker image contents, Lean toolchain pinning rules, and never-in-image secret policy. [email-strategy-baseline.md](email-strategy-baseline.md) defines the MVP decision to keep app email out of scope for now, split support mail from transactional mail, and use Resend as the default later outbound provider. [operations-baseline.md](operations-baseline.md) records the service ownership, hostname, and secret model that the current roadmap relies on. [data-baseline.md](data-baseline.md) captures the first database entity and branch strategy decisions. [testing-baseline.md](testing-baseline.md) defines the MVP test stack choice and required testing methodology. [project-management.md](project-management.md) explains how scoping issues and execution issues are meant to move through GitHub. The accepted implementation stack decision is recorded in [adrs/0001-mvp-implementation-stack.md](adrs/0001-mvp-implementation-stack.md) and the hosting choice in [adrs/0002-mvp-hosting-decision.md](adrs/0002-mvp-hosting-decision.md).
